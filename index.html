عملیات و مدیریت لجستیک حمل و نقل

هدف

در این بخش، شما بر اساس موجودیت هایی که در مینی‌پروژه ۱ طراحی کردید، ادامه خواهید داد. شما منطقی را برای مدیریت این موجودیت‌ها پیاده‌سازی خواهید کرد که بر موارد زیر تمرکز دارد:

  • ماندگاری داده‌ها (ساده‌شده): ذخیره و بازیابی داده‌های موجودیت.
  • عملیات CRUD: ایجاد، خواندن، به‌روزرسانی و حذف موجودیت‌ها.
  • جستجو و فیلتر: یافتن کارآمد موجودیت‌های خاص بر اساس معیارها.
  • شبیه‌سازی لجستیک (پایه): شبیه‌سازی حرکت و تخصیص بسته‌ها.
  • مدیریت خطا: مدیریت قوی عملیات نامعتبر و موارد خاص.

مفاهیم اصلی برای به کارگیری

  • ساختارهای داده: استفاده از لیست‌ها، دیکشنری‌ها یا مجموعه‌ها برای ذخیره‌سازی و بازیابی کارآمد.
  • کپسوله‌سازی: بهبود بیشتر متدهای کلاس برای مدیریت وضعیت داخلی.
  • چندریختی: استفاده از رابط‌های مشترک کلاس‌های پایه شما در صورت لزوم.
  • مدیریت خطا: پیاده‌سازی بلوک‌های try-except و استثنائات سفارشی.
  • تفکر الگوریتمی: طراحی منطق جستجو و مدیریت کارآمد.

بخش 1: آماده‌سازی محیط

قبل از شروع، باید کلاس‌هایی را که در مینی‌پروژه ۱ تعریف کرده‌اید، وارد کنیم. تمام تعاریف کلاس‌های تکمیل‌شده خود (City، Package و زیرکلاس‌های آن، Container و زیرکلاس‌های آن، Truck و زیرکلاس‌های آن) را در سلول زیر کپی و جایگذاری کنید. این کار تضمین می‌کند که تمام موجودیت‌های شما برای عملیات جدیدی که پیاده‌سازی خواهید کرد، در دسترس هستند.

کد کلاس‌های مینی‌پروژه ۱

نکته:

تمام تعاریف کلاس‌های خود را از مینی‌پروژه ۱ در اینجا جایگذاری کنید.

بخش 2: کلاس LogisticsManager

برای متمرکز کردن عملیات و مدیریت مجموعه‌های موجودیت‌های لجستیکی خود، یک کلاس LogisticsManager ایجاد خواهیم کرد. این کلاس به عنوان مغز سیستم ما عمل می‌کند و تمام عملیات CRUD، قابلیت‌های جستجو و منطق پایه لجستیک را مدیریت می‌کند.

1. کلاس LogisticsManager

توضیحات: تمام اشیاء City، Package، Container و Truck را در سیستم مدیریت می‌کند. باید مجموعه‌هایی از این اشیاء را نگهداری کند.

  • ویژگی‌های اجباری:
    • cities (دیکشنری): اشیاء City را بر اساس id آنها ذخیره می‌کند.
    • packages (دیکشنری): اشیاء Package را بر اساس id آنها ذخیره می‌کند.
    • containers (دیکشنری): اشیاء Container را بر اساس id آنها ذخیره می‌کند.
    • trucks (دیکشنری): اشیاء Truck را بر اساس id آنها ذخیره می‌کند.
  • متدهای اجباری:
    • __init__ (سازنده): تمام دیکشنری‌های داخلی را مقداردهی اولیه می‌کند.
    • add_city(city): یک شیء City را به دیکشنری cities اضافه می‌کند. اطمینان حاصل کنید که ID منحصر به فرد است. در صورت موفقیت True و در صورت وجود ID قبلاً False را برمی‌گرداند.
    • get_city(city_id): یک شیء City را بر اساس ID آن بازیابی می‌کند. شیء City یا None را در صورت عدم یافتن برمی‌گرداند.
    • update_city(city_id, new_name=None, new_details=None): نام یا جزئیات یک شهر را به‌روزرسانی می‌کند. در صورت موفقیت True و در صورت عدم یافتن شهر False را برمی‌گرداند.
    • delete_city(city_id): یک شیء City را حذف می‌کند. مهم: قبل از حذف یک شهر، اطمینان حاصل کنید که هیچ بسته‌ای در حال حاضر به آن به عنوان مبدا یا مقصد اشاره نمی‌کند. در صورت وجود، از حذف جلوگیری کرده و False را برمی‌گرداند. در صورت حذف موفقیت‌آمیز True و در غیر این صورت False را برمی‌گرداند.
    • add_package(package): یک شیء Package را به دیکشنری packages اضافه می‌کند. اطمینان حاصل کنید که ID منحصر به فرد است و شهرهای مبدا و مقصد آن در مجموعه cities مدیر موجود هستند. در صورت موفقیت True و در غیر این صورت False را برمی‌گرداند.
    • get_package(package_id): یک شیء Package را بر اساس ID آن بازیابی می‌کند. شیء Package یا None را در صورت عدم یافتن برمی‌گرداند.
    • update_package_destination(package_id, new_destination_city_id): مقصد یک بسته را به‌روزرسانی می‌کند. اطمینان حاصل کنید که new_destination_city_id با یک شهر موجود مطابقت دارد. در صورت موفقیت True و در غیر این صورت False را برمی‌گرداند.
    • delete_package(package_id): یک شیء Package را حذف می‌کند. مهم: اگر بسته در حال حاضر در یک کانتینر یا روی یک کامیون بارگیری شده باشد، ابتدا باید تخلیه شود. در صورت حذف موفقیت‌آمیز True و در غیر این صورت False را برمی‌گرداند.
    • add_container(container): یک شیء Container را به دیکشنری containers اضافه می‌کند. اطمینان حاصل کنید که ID منحصر به فرد است. در صورت موفقیت True و در صورت وجود ID قبلاً False را برمی‌گرداند.
    • get_container(container_id): یک شیء Container را بر اساس ID آن بازیابی می‌کند. شیء Container یا None را در صورت عدم یافتن برمی‌گرداند.
    • delete_container(container_id): یک شیء Container را حذف می‌کند. مهم: اگر کانتینر در حال حاضر روی یک کامیون بارگیری شده باشد، ابتدا باید تخلیه شود. در صورت حذف موفقیت‌آمیز True و در غیر این صورت False را برمی‌گرداند.
    • add_truck(truck): یک شیء Truck را به دیکشنری trucks اضافه می‌کند. اطمینان حاصل کنید که ID منحصر به فرد است. در صورت موفقیت True و در صورت وجود ID قبلاً False را برمی‌گرداند.
    • get_truck(truck_id): یک شیء Truck را بر اساس ID آن بازیابی می‌کند. شیء Truck یا None را در صورت عدم یافتن برمی‌گرداند.
    • delete_truck(truck_id): یک شیء Truck را حذف می‌کند. مهم: اگر کامیون دارای بسته یا کانتینر بارگیری شده باشد، ابتدا باید تخلیه شوند یا عملیات ممنوع شود. در صورت حذف موفقیت‌آمیز True و در غیر این صورت False را برمی‌گرداند.
    • load_package_to_container(package_id, container_id): بسته و کانتینر را پیدا می‌کند، سپس سعی می‌کند بسته را با استفاده از متد add_package کانتینر در آن بارگیری کند. در صورت موفقیت True و در غیر این صورت False را برمی‌گرداند (به دلیل عدم سازگاری، محدودیت‌های وزن/تعداد و غیره).
    • unload_package_from_container(package_id, container_id): بسته و کانتینر را پیدا می‌کند، سپس سعی می‌کند بسته را از کانتینر تخلیه کند. در صورت موفقیت True و در غیر این صورت False را برمی‌گرداند.
    • load_item_to_truck(item_id, truck_id): این متد باید هم بسته‌ها و هم کانتینرها را مدیریت کند. باید تشخیص دهد که item_id به یک Package یا Container اشاره دارد، سپس متد بارگیری مناسب را روی شیء Truck فراخوانی کند (یا load_package برای NormalTruck یا load_container برای ContainerTruck). در صورت موفقیت True و در غیر این صورت False را برمی‌گرداند.
    • unload_item_from_truck(item_id, truck_id): مشابه load_item_to_truck، این متد باید نوع آیتم را تشخیص داده و متد تخلیه مناسب را فراخوانی کند. در صورت موفقیت True و در غیر این صورت False را برمی‌گرداند.
    • find_packages_by_destination(city_id): لیستی از اشیاء Package را که مقصد آنها شهر مشخص شده است، برمی‌گرداند. در صورت عدم یافتن، یک لیست خالی برمی‌گرداند.
    • find_containers_with_fragile_packages(): لیستی از اشیاء Container را که حداقل یک FragilePackage دارند، برمی‌گرداند. این متد باید از طریق کانتینرها و سپس از طریق بسته‌های داخل آن کانتینرها پیمایش کند.
    • get_trucks_carrying_container_type(container_type): لیستی از اشیاء Truck را که در حال حاضر حداقل یک کانتینر از نوع مشخص شده (مانند "FreezerContainer"، "FragileContainer") حمل می‌کنند، برمی‌گرداند.
    • get_all_packages_in_transit(): لیستی از تمام بسته‌هایی را که در حال حاضر در هر کانتینر یا روی هر کامیون بارگیری شده‌اند، برمی‌گرداند.

بخش 3: نمایش قابلیت‌های کلاس جدید

در این بخش، شما LogisticsManager را نمونه‌سازی کرده و از متدهای آن برای نمایش تمام قابلیت‌هایی که پیاده‌سازی کرده‌اید، استفاده خواهید کرد. برای هر عملیات، پیام‌های واضحی را چاپ کنید که موفقیت یا عدم موفقیت و دلیل آن را نشان دهد.

1. مقداردهی اولیه

یک نمونه از LogisticsManager خود را ایجاد کنید.

2. نمایش عملیات CRUD شهر

  • چندین شیء City را اضافه کنید.
  • سعی کنید شهری را با ID تکراری اضافه کنید.
  • یک شهر را با ID (هم موجود و هم غیر موجود) بازیابی کنید.
  • جزئیات یک شهر را به‌روزرسانی کنید.
  • سعی کنید شهری را که بسته‌هایی با آن مرتبط هستند حذف کنید (باید شکست بخورد).
  • شهری را که هیچ وابستگی ندارد حذف کنید.

3. نمایش عملیات CRUD بسته

  • انواع مختلفی از اشیاء Package (فریزر، شکننده، عمومی) را اضافه کنید، اطمینان حاصل کنید که شهرهای مبدا/مقصد آنها در کلاس موجود هستند.
  • سعی کنید بسته‌ای را با شهرهای غیر موجود اضافه کنید.
  • یک بسته را با ID بازیابی کنید.
  • مقصد یک بسته را به‌روزرسانی کنید.
  • سعی کنید بسته‌ای را که در حال حاضر در یک کانتینر/کامیون بارگیری شده است حذف کنید (ابتدا باید شکست بخورد، سپس پس از تخلیه موفق شود).

4. نمایش عملیات CRUD کانتینر

  • چندین شیء Container را اضافه کنید.
  • سعی کنید کانتینری را با ID تکراری اضافه کنید.
  • یک کانتینر را با ID بازیابی کنید.
  • سعی کنید کانتینری را که در حال حاضر روی یک کامیون بارگیری شده است حذف کنید (ابتدا باید شکست بخورد، سپس پس از تخلیه موفق شود).

5. نمایش عملیات CRUD کامیون

  • اشیاء NormalTruck و ContainerTruck را اضافه کنید.
  • سعی کنید کامیونی را با ID تکراری اضافه کنید.
  • یک کامیون را با ID بازیابی کنید.
  • سعی کنید کامیونی را با آیتم‌های بارگیری شده حذف کنید (باید شکست بخورد).

6. نمایش منطق بارگیری/تخلیه بسته به کانتینر

  • بسته‌های مناسب را در FreezerContainer و FragileContainer بارگیری کنید.
  • سعی کنید بسته‌های ناسازگار (نوع اشتباه، خیلی سنگین، بیش از حد زیاد، دمای نامناسب) را در کانتینرهای تخصصی بارگیری کنید و نتایج را چاپ کنید.
  • get_current_weight() و get_current_package_count() را برای کانتینرها نمایش دهید.
  • بسته‌ها را از کانتینرها تخلیه کنید.
  • سعی کنید یک بسته غیر موجود را تخلیه کنید.

7. نمایش منطق بارگیری/تخلیه آیتم به کامیون

  • کانتینرها را روی ContainerTruck بارگیری کنید.
  • بسته‌ها را روی NormalTruck بارگیری کنید.
  • تلاش برای بارگیری بیش از حد کامیون‌ها یا بارگیری اقلام با نوع اشتباه (به عنوان مثال، بارگیری مستقیم یک Package روی ContainerTruck، یا یک Container روی NormalTruck) را نمایش دهید.
  • get_current_payload_weight() را برای کامیون‌ها نمایش دهید.
  • آیتم‌ها را از کامیون‌ها تخلیه کنید.

این سند برای پروژه کوچک عملیات و مدیریت لجستیک حمل و نقل ایجاد شده است.
